P0-A 原型

飞书多维表格 × 微信公众号桥接工具

先打通飞书 CLI 用户态授权、Base 绑定和数据读取，微信侧配置预留到下一步接入。

0. 当前执行环境

确认现在真正运行后端和 lark-cli 的服务器环境，适合跨设备远程开发时排查。

后端主机 未检查
Node.js 未检查
lark-cli 未检查
授权用户 未检查

1. 飞书 CLI 状态

确认本机后端能找到并执行 lark-cli。

2. 引导式配置初始化

用于创建/绑定飞书应用。该流程可能打开浏览器或输出操作提示，需要用户按飞书 CLI 引导完成。

3. 用户扫码授权

使用 split-flow：先发起授权，用户完成后，再用 device_code 完成登录。

4. 绑定多维表格

输入已有 Base 链接，解析出 base_token、table_id、view_id。

5. 模板表与数据读取

可先创建「推送草稿表」，再读取字段和记录。记录读取会强制使用 JSON 格式。

6. 微信侧配置占位

P0-A 先不调用微信 API，只收集字段结构，下一步接入 access_token、素材和草稿。

输出

所有接口统一返回 ok/data/error，方便排查 CLI 权限、授权和 Base 参数问题。
